Pick a Status Reporting Pattern

Balancing real-time visibility with implementation simplicity

A

Shared Status Files

Each agent writes its progress to status-backend.json in the shared volume. Orchestrator polls these files.

  • ✅ No extra dependencies (file-based)
  • ❌ Slight delay due to polling
B

Git-Driven Status

Agents report progress via Git Commit Messages or by updating a PROGRESS.md in their own branch. Orchestrator monitors Git.

  • ✅ Traceable history of work
  • ❌ Harder to parse for real-time dashboard
C

Real-time Webhook

Agents send a small HTTP POST to the Orchestrator's dashboard API whenever their state changes.

  • ✅ Instant dashboard updates
  • ❌ Requires agents to have network access to Hub
