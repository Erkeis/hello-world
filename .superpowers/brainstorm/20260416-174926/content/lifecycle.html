Which "Intent Delivery" method feels more efficient?

Considering your concern about resource management vs. operational overhead

A

Global Intent Filtering

Orchestrator writes one intent.json. Each agent (frontend, backend, etc.) reads it and filters for tasks relevant to its role.

  • ✅ Simple orchestrator logic
  • ❌ Agents spend tokens reading irrelevant tasks
B

Targeted Intent Injection

Orchestrator writes specific files (intent-frontend.json, etc.). Each agent only sees what it needs to do.

  • ✅ Minimal agent token waste
  • ❌ More complex orchestrator logic
