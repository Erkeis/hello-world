## [2026-04-18] Phase 3: Intelligence & Autonomy Decisions

### 9. Data Integrity: JSONL Source Strategy
- **Decision:** **JSONL-first, Markdown-view**.
- **Rationale:** JSONL (Newline Delimited JSON) provides O(1) append performance and superior corruption resilience compared to YAML or single JSON. It allows the Orchestrator to "snipe" context via efficient tail-reads. Markdown documents for human review will be auto-generated from this source.

### 10. Concurrency: Atomic State & Kernel Events
- **Decision:** **Atomic Write & inotify**.
- **Rationale:** Prevents race conditions during status updates in `intent.json`. Use "Write to temp -> Atomic Rename" pattern. Subagents will use `inotifywait` for near-zero latency reactions instead of CPU-intensive polling.

### 11. Lifecycle: Docker Native Session Reset
- **Decision:** **Force Recreate / Restart via Hub**.
- **Rationale:** Simplifies session purification. Instead of injecting complex "exit" commands into interactive agent processes, the Hub will use `docker compose restart` to ensure a 100% clean memory and state flush between tasks.

### 12. Workflow: Risk-based Tiered Approval (P0)
- **Decision:** **3-Tier Approval Logic**.
- **Rationale:** Solves the "CEO Bottleneck."
    - **Tier 1 (Low):** Read-only tasks (log analysis, research) -> **Auto-Approve**.
    - **Tier 2 (Med):** Non-destructive actions (linting, tests, typos) -> **Flag & Notify** (Implicit approval).
    - **Tier 3 (High):** Destructive/Structural changes (logic, architecture) -> **Manual Gate** (Explicit approval required).
