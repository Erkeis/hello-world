# Decision Log: Agentic Environment Farm

This document records the key architectural and technical decisions made during the evolution of the Agentic Environment Farm.

## [2026-04-17] Phase 2: Agentic Neural Network & Scaling Decisions

### 1. Log Streaming Scalability (SSE)
- **Decision:** **Unified Log Bus (1B)**
- **Rationale:** 12 individual SSE streams would exhaust file handles and client connection limits. Aggregating all logs into a single stream with client-side filtering ensures scalability up to 100+ agents.

### 2. Git Merge Strategy & Conflict Handling
- **Decision:** **Fail-Fast with Notifications (2A)**
- **Rationale:** Automated rebasing is risky and may lose context. Freezing the agent in a `BLOCKED:CONFLICT` state and notifying the User (via Beep/Messenger) allows for safe, high-priority human intervention.

### 3. Agent Factory Interface
- **Decision:** **CLI-first with Future API (3A/3B Hybrid)**
- **Rationale:** Prioritize a stable SSH-based CLI for Phase 2 infrastructure generation. Implementation of a seamless Dashboard-driven API is deferred to Phase 3 to ensure functional stability first.

### 4. Dashboard State Management
- **Decision:** **Isolated Agent Contexts (4B)**
- **Rationale:** Managing 12 concurrent log streams in a single global state would cause UI-killing re-renders. Using React Context or individual state hooks ensures only the active cell updates.

### 5. Config Management (Shared Paths)
- **Decision:** **Centralized Config via ENV (5B)**
- **Rationale:** Adheres to DRY principles. Prevents hardcoding paths like `shared-context` in multiple files, making the system "Cloud-Ready" for future migration.

### 6. SSE Lifecycle & Resource Management
- **Decision:** **Explicit Lifecycle Cleanup (6A)**
- **Rationale:** Ensures all Chokidar file watchers and system resources are explicitly closed when an SSE connection is dropped, preventing memory leaks on the Ubuntu server.

### 7. Concurrency Verification
- **Decision:** **Mandatory Parallel Merge Regression Test**
- **Rationale:** Given the high risk of Git index locks with 12 agents, a dedicated test case that forces and resolves conflicts is required before shipping Phase 2.

### 8. Dashboard Log Performance
- **Decision:** **Log Windowing / Capping (8B)**
- **Rationale:** Limits log history to 200 lines per agent in the grid view to prevent DOM bloat and maintain UI responsiveness during high-frequency agent activity.
