## Problem Statement
A solo developer is losing 3-4 hours daily to repetitive environment setup and manual "context propagation" (copy-pasting intent across multiple LLM sessions). Daily token quotas are wasted because the human remains the synchronous bottleneck between the model and the environment.

## Demand Evidence
Real-world daily waste: 4 hours of manual toil. The developer's productivity is currently throttled by their own manual coordination of isolated agents.

## Target User & Narrowest Wedge
**Target User:** The Solo Developer / "Harness Engineer" who needs to manage 12+ agents concurrently.
**Narrowest Wedge:** A **Context Sync Hook**—a headless Ubuntu server running Docker containers that automatically pull shared intent from a central Git repository via hooks. This eliminates the first hour of copy-paste work by tomorrow.

## Core Architecture: Simplified Hierarchy
To reduce orchestration overhead, the environment farm utilizes a two-layer simplified hierarchy:
1. **Orchestrator (HR/CEO):** Manages high-level concepts, cross-container resource allocation, and overall project direction.
2. **Worker (Containerized Team):** A single unified layer for execution. Internal logic (e.g., calling `@generalist` or spawning sub-tasks) is managed autonomously within the container. 
   - *Note: Previous distinctions between "Lead" and "Worker" layers are collapsed into this single execution unit.*

## Identification & Naming Strategy (Hybrid Layer)
Containers use a naming convention that balances invariant identity with session-specific flexibility:
- **Convention:** `UniqueName:Port [SessionRole]`
- **Example:** `sec-forge:8002 [sec-lead]`
  - `sec-forge`: Invariant identity (The "Forge" container for security tasks).
  - `8002`: Routing and connectivity identifier.
  - `sec-lead`: Variable role assigned for the current session.

## Git Identification & Convention
To maintain an agile and low-friction environment, the farm uses a **Shared Account + Prefix Convention** strategy:
- **Account:** Shared Git accounts are used to minimize setup overhead.
- **Commit Convention:** `agent/name[role]: message`
  - Example: `agent/sec-forge[sec-lead]: fix authentication flow`
- This ensures that every contribution is immediately traceable to a specific agent identity and its current role without complex account management.

## Agile Operational Flow (HR Flow)
The interaction between the Orchestrator and the Workers follows a lean HR-style reporting flow:
1. **Concept Delivery:** The Orchestrator delivers a high-level Concept or Mission to the Container.
2. **Internal Organization:** The Container autonomously organizes its internal team and task breakdown.
3. **HR Report:** The Container reports its internal structure and plan back to the Orchestrator for review.
4. **Indirect Management:** Progress and alignment are managed indirectly through Git branches, allowing for asynchronous validation and high-concurrency operations.

## Constraints
- Windows server hardware (transitioning to headless Linux/Ubuntu).
- Multi-container Docker isolation (12+ agents).
- Resource efficiency (headless server preferred).
- Visualization (4x3 grid for visibility).

## Success Criteria
- Time spent on manual context-passing reduced from 4 hours to <30 minutes.
- 12 containers running concurrently with zero resource starvation.
- All agent activity visible in a single 4x3 grid view using the `Name:Port [Role]` identifier.

## Dependencies
- Transitioning server OS to Linux (Ubuntu Server 24.04).
- Docker Engine & docker-compose.
- Node.js for the Dashboard UI.
