# Automation Candidate Buffer (Phase 3 Post-Meeting - REFINED)

이 문서는 Phase 1~3의 개발 및 운영 경험을 바탕으로, 반복적인 수동 조작을 줄이고 시스템의 자율성을 높이기 위해 'Skill' 또는 'Script'화할 수 있는 후보들을 정리한 사전 검토안입니다.

## 🤖 스킬/스크립트화 후보 리스트

### 1. [Skill] Agent Lifecycle Manager
- **레이어:** **Layer 1 (Global/Root)**
- **대상 작업:** 워크트리 생성, 컨테이너 재생성, `intent.json` 초기화, `.blocked` 파일 정리.
- **메타 설명:** 에이전틱 OS의 물리적 환경을 정화하는 '관리용 하네스'. 프로젝트 수출(Export) 시 제외되며 호스트의 관리 역량으로 남음.

### 2. [Script] Context Sniper Eval Test
- **레이어:** **Layer 1 (Global/Root)**
- **대상 작업:** `#tag` 기반 추출이 결정 로그의 변경에도 여전히 정확한지 검증.
- **메타 설명:** 데이터 무결성을 감시하는 '거버넌스 도구'. 인메모리 캐시의 신뢰도를 보장함.

### 3. [Skill] Agile HR Dispatcher
- **레이어:** **Layer 2 (Application-Specific)**
- **대상 작업:** 큰 컨셉을 12개 에이전트의 타겟팅된 `intent.json`으로 분해하여 자동 전송.
- **메타 설명:** 대시보드 UI 앱의 '지능형 비즈니스 로직'. 프로젝트 수출 시 포함되어 제품의 자율적 동작을 정의함.

### 4. [Tool] Git-Harness Sanity Checker
- **레이어:** **Layer 1 (Global/Root)**
- **대상 작업:** `.gitignore` 누락, `.agent` 폴더 유출, 태스크 단위 커밋 준수 여부 자동 감시.
- **메타 설명:** 이력 관리의 무결성을 강제하는 '엔지니어링 수호자'. 사용자님이 지적하신 '운영의 디테일'을 자동화함.
