# Cognitive Mode Switching Guide

## 1. Background: Why Differentiate Modes?

In AI collaboration, 'Philosophy' and 'Structure' act as different engines.
Applying the same cognitive approach to all tasks leads to wasted efficiency or stifled innovation.

## 2. Defining the Two Modes

### Structural Mode

| Field | Content |
|:---|:---|
| **Purpose** | Precise tool selection, token saving, consistent output, error prevention |
| **Metaphor** | A well-paved highway ??reaching the destination fastest with minimal fuel consumption |
| **Operation** | Deterministic ??rule-based, pattern matching, optimization |
| **Core Question** | "What is the most efficient method?" |

### Philosophical Mode

| Field | Content |
|:---|:---|
| **Purpose** | Breaking existing patterns, new perspectives, root-cause resolution, discovering innovation in inefficiency |
| **Metaphor** | Uncharted territory ??the "Why this path?" is more important than the shortest distance |
| **Operation** | Heuristic ??analogy, divergence, multi-perspective exploration |
| **Core Question** | "What value should this provide to the user?" |

## 3. Mode Decision Matrix

| Task Type | Mode | Rationale |
|:---|:---|:---|
| Data Parsing / API Calls | Structure | Deterministic results. No divergence needed |
| Workflow Execution | Structure | Precision and order are paramount |
| Context Routing / Metadata Design | Structure | Efficiency is the primary value |
| Code Generation (Clear Specs) | Structure | Direct path from spec to implementation |
| Initial Planning / Drafting | Philosophy | Refinement of direction and value needed |
| Persona / Tone Setting | Philosophy | Requires human-like intuition and context interpretation |
| Architecture Design (New System) | Philosophy | Creating structures not found in existing patterns |
| UX Enhancement | Philosophy | Predicting user emotions and behavior |
| Root Cause Analysis | Philosophy | Must look at the essence, not just symptoms |

## 4. Execution Principles

1. **Default to Structural Mode.** Most agent tasks are execution-oriented.
2. **Enter Philosophical Mode explicitly.** Switch only when the task asks "Why?".
3. **Separate steps for mixed tasks.** Direction setting (Philosophy) ??Execution (Structure).
4. **Confirm with user if uncertain.** The wrong mode creates loss on both sides.

## 5. Summary Formula

```
Implementation = Delegate to Structure for Deterministic AI behavior
Direction      = Delegate to Philosophy for Heuristic human-like thinking
```

## 6. Origin

This guide was derived from insights into "Efficiency of Structure vs Creativity of Divergence" captured during the design of Lean Metadata-Driven Context Routing.
